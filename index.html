Quick Start

Install the plugin the same way your buyers will

  1. Open Epic Games Launcher and go to your FAB Library or Vault.
  2. Find Multiplayer Chat System, choose the Unreal Engine version you want to use, then download or install that build.
  3. Add the plugin to your target project from the Launcher library and open the project in Unreal.
  4. Enable the plugin if Unreal asks for activation or restart.
  5. Make your GameState inherit from AChatSystemGameState.
  6. Make your PlayerController inherit from AChatSystemPlayerController.
  7. Open Project Settings -> Plugins -> Multiplayer Chat System -> Chat Integration.
  8. Use the built-in selector or Auto-Detect to find your playable Pawn or Character Blueprint.
  9. Press Apply Chat To Pawn to add UChatComponent and UChatBubbleComponent to that blueprint.
  10. Create your chat input buttons or action mappings before testing in Play.
  11. Bind OpenChat to FocusChatInput() and CloseChat to ReleaseChatInputFocus().
  12. Recommended default buttons: Enter to open or send chat, and Escape to close chat.

Runtime classes

GameState -> AChatSystemGameState
PlayerController -> AChatSystemPlayerController
Pawn / Character -> UChatComponent + UChatBubbleComponent

Input expectation

OpenChat  -> FocusChatInput()
CloseChat -> ReleaseChatInputFocus()
Enter -> submit message
Escape -> exit chat mode (if enabled)

Input mappings are not auto-created by the plugin. The recommended flow is to define your chat buttons first, then test focus, send, and close behavior in Play.

Commands

Supported slash commands

Command Purpose
/help, /?, /commands Shows the available chat commands and emotes.
/g, /global, /s, /say Sends a public message to the shared feed.
/y, /yell Sends a proximity-based message using the configured yell range.
/w Name message, /whisper Mirrors a private message between sender and target only.
Text / animation emotes Uses the built-in emote tables configured in the plugin.

Whispers are resolved deterministically: no match returns an error, one match delivers normally, and duplicate names are treated as ambiguous instead of choosing a target at random.

Settings Guide

What each editor block controls

The screenshots below follow the same order as the in-editor workflow. Use them as a quick visual reference while tuning the HUD, messages, quick settings gear button, integration, and world chat bubbles.

Screenshot 1

Theme, frame image, and HUD layout

This block defines the overall look of the chat window before you start tuning the smaller details.

  • Theme switches between built-in presets or your current custom override.
  • HUD Frame Image selects the PNG frame loaded from Resources/Widget.
  • Overall Scale scales the full chat HUD at once.
  • Widget Size changes the total width and height of the chat container.
  • Positioning Mode decides how placement is calculated, including anchor-based setups.
  • Feed Placement and Input Placement control where the message list and input bar sit inside the frame.
  • Feed Padding and Input Padding add breathing room around those areas.
  • Feed Background Opacity and Input Field Opacity control how visible those inner surfaces are.
  • Chat Frame In Front decides whether the frame art renders over the message feed.
  • Collapsed Box Scale and Hide Feed When Collapsed shape the minimized state.

Screenshot 2

Message cards, input behavior, and startup text

This area controls readability, how the chat feels while typing, and what players see when a session starts.

  • Entry Spacing and Accent Width tune the rhythm and left accent bar of each message row.
  • Content Padding adds internal spacing inside each entry card.
  • Entry Background Opacity controls how strong the message card surface appears.
  • Channel Badge Opacity makes the channel tags stronger or more subtle.
  • Sender Text Color and Timestamp Text Color let you push readability or art direction.
  • Open Chat on Enter, Keep Focus After Send, and Close Chat on Escape shape the basic typing loop.
  • Enable Input History and Enable Autocomplete enable quality-of-life chat behavior.
  • Animate Chat HUD and Fade Minimized Chat When Idle control motion and idle fading.
  • Minimized Idle Delay and Minimized Idle Opacity define when the minimized HUD fades and how far it fades.
  • Show Welcome Message in Play, System Sender Name, and Welcome Message Lines configure the startup system feed.

Screenshot 3

Quick settings button, character integration, and bubble body

This block connects the plugin to your playable character and defines how the world bubble behaves in 3D space.

  • Custom Icon PNG lets you swap the gear icon for your own image.
  • Button Offset moves the quick settings button inside the HUD frame.
  • Button Size changes the button dimensions without resizing the entire chat.
  • Background Color and Background Opacity control the gear button plate behind the icon.
  • Play Character / Pawn, Auto-Detect, and Refresh define which blueprint receives the chat components.
  • Use Selected Pawn in Play applies the currently selected character choice at runtime.
  • Apply Chat To Pawn adds the plugin components, while Remove Chat From Pawn removes them explicitly.
  • Enable Chat Bubble turns world bubbles on or off globally.
  • World Scale, Height Offset, Visible Duration, and Fade Duration define size, position, and lifetime.
  • Max Width, Padding, Background Color, Background Opacity, and Corner Radius shape the bubble body itself.

Screenshot 4

Bubble text styling

This final block controls how the player name and message text are drawn inside each world bubble.

  • Show Speaker Name toggles the name label on or off.
  • Tint Message By Type lets different chat types influence the bubble text color treatment.
  • Speaker Font Size controls the name size.
  • Message Font Size controls the body text size.
  • Speaker Color sets the base name color when type tinting is not overriding it.

Architecture

How message delivery is structured

Input Player controller or fallback component UI captures text.
Authority UChatComponent validates commands, ranges, length, and whisper resolution on the server.
History AChatSystemGameState stores public message history with replicated sequence IDs.
Presentation HUD feed receives live and replay-safe entries while bubbles only trigger for live messages.

The plugin keeps the current public API intact while using event-based delivery to avoid fragile polling paths.

FAQ

Common integration questions

Can I style the HUD without touching C++?

Yes. The plugin exposes theme, typography, layout, bubble, and quick settings button controls through Project Settings and the live designer window.

Can I use my own frame art?

Yes. The HUD theme supports PNG frame images, and the quick settings button can use a custom icon image as well.

Does the plugin support late joiners?

Yes. Public history is replayed through the replicated GameState history, while live-only bubble events remain protected from history replays.

Can I give each character a custom chat name?

Yes. The Chat Component includes a configurable display name that is used by chat, whispers, connected-name lists, emotes, and world bubbles.
